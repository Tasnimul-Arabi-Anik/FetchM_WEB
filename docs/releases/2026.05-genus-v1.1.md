# FetchM Web 2026.05-genus-v1.1

## Scope

This release freezes the current FetchM Web production state after the genus-level metadata-standardization release and the follow-up web/admin hardening work. It should be used as the rollback/reference point for future UI, security, admin, and metadata-refresh changes.

## Changes Since `2026.05-genus-v1`

- Added user-facing and admin operations guides:
  - `docs/USER_GUIDE.md`
  - `docs/ADMIN_OPERATIONS_GUIDE.md`
- Added admin job analytics and operational monitoring improvements.
- Added admin audit bundle download documentation and UI access.
- Polished the sign-in page into a centered, unified board with clearer text and tighter spacing.
- Fixed BioSample cache-hit classification so metadata fetch status is reported correctly.
- Preserved the deterministic metadata standardization rule set; no broad rewrite was performed after the final production audit.

## Operational Notes

- Keep standardization deterministic and rule-file based.
- Do not add BGE/embedding output directly into production standardized metadata. Embeddings may be used only to cluster review queues.
- Continue low-frequency host/source-like curation as maintenance, not as a blocking release issue.
- Before future production changes, create a new release note and tag rather than moving this tag.
