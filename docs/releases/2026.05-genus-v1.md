## Platform Changes

- Login/register/reset UX now includes clearer purpose text and accessibility labels.
- New passwords require at least 10 characters with at least one letter and one number.
- Sessions have a configurable lifetime through `FETCHM_WEBAPP_SESSION_HOURS`.
- CSRF validation is enabled for POST requests.
- Login and password-reset endpoints have per-IP rate limiting.
- Admin overview exposes an access/security posture panel.
- Admin POST actions and selected user actions are recorded in an append-only audit log.
- Upload validation is restricted to expected `.tsv` or `.csv` files with a configurable size ceiling.
- Container image runs as a non-root `fetchm` user.

## Remaining Maintenance Backlog

- Low-frequency host/source-like curation remains a maintenance task, not a blocking release issue.
- Multi-factor authentication, role-based admin permissions, API tokens, and quota management are deferred.
- BGE/embedding should remain a review-assistant only; production standardization remains deterministic and auditable.
